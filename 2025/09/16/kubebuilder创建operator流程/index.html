kubebuilder创建operator流程

开发流程详解（以 Kubebuilder 为例）

我们将通过一个简单的例子来说明：开发一个名为 Guestbook 的 Operator。用户创建一个 Guestbook 资源时，Operator 会自动为我们部署一个 Redis 主实例和一个 Redis 从实例。

阶段一：环境准备

  1. 安装必备工具：
    • go： Golang 开发环境。
    • docker： 用于构建 Operator 的容器镜像。
    • kubectl： 与 Kubernetes 集群交互。
    • kustomize： Kubebuilder 用来配置部署 YAML 的工具（通常包含在 kubebuilder 安装包中）。
    • **kubebuilder**： 核心框架工具。跟随官方文档安装。
  2. 准备一个 Kubernetes 集群：
    • 可以使用 Minikube、Kind（Docker in Docker）或任何开发测试集群。

总结与核心要点

步骤 关键命令/文件 目的
初始化 kubebuilder init 创建项目脚手架和入口文件
创建API kubebuilder create api 生成 CRD 类型定义和控制器骨架
定义API api/<version>/<kind>_types.go 定义 CR 的 Spec 和 Status 结构
生成代码 make generate, make manifests 根据类型定义生成代码和 CRD YAML
实现逻辑 controllers/<kind>_controller.go 在 Reconcile 函数中编写业务逻辑
安装CRD make install 将 CRD 安装到集群
构建部署 make docker-build, make docker-push, make deploy 将 Operator 本身部署到集群
测试 kubectl apply -f config/samples/ 创建 CR 实例，触发 Operator 工作
